# Product Messaging & Design Philosophy

## Purpose
This document defines how LeadScorer speaks and how LeadScorer screens are designed.

## Product Message
- Problem: CRM is full, but priority is unclear.
- Promise: LeadScorer tells the team who to contact first, today.
- Method: Import leads, score transparently, act from one queue.
- Next step language: always action-first (`Try in Dashboard`, `Log In`, `Start tour`).

## YC-Style Clarity Rules
- Start with user value, not technology.
- One clear statement per section.
- Use plain words; no infrastructure jargon.
- Every section must answer: `Why should I care now?`

## Don't Make Me Think Rules
- Users should understand each block in 5 seconds.
- One primary action per screen context.
- Labels must be literal (`Hot`, `Warm`, `Cold`, `Contacted`, `Snoozed`).
- Explanations must be short and visible at decision points.

## Lean Product Thinking Rules
- Keep only features that reduce time-to-priority.
- Prefer deterministic logic over black-box behavior in prototype mode.
- If a UI block does not support scoring trust or queue action speed, remove it.
- Prototype claims must stay honest (`session-only`, `no backend persistence`).

## Tone & Writing Constraints
- No fluff
- No repetition
- No vague language
- No generic startup phrasing
